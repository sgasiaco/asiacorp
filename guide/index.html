🟢 GoDaddy DNS Setup for GitHub Pages

Configure your GoDaddy DNS records to point your custom domain at GitHub Pages.

🔧 Registrar: GoDaddy
🚀 Host: GitHub Pages
⏱ Time: ~10 min (+ DNS propagation)
1
Delete the Default "www" CNAME Record
GoDaddy DNS Management

GoDaddy pre-creates a CNAME record for www pointing back to your own domain. You must delete this before adding the GitHub one — otherwise it will conflict.

2
Check for & Remove Any Existing "A" Records
GoDaddy DNS Management — clear conflicts before adding new ones

If your domain was previously pointing to a web host, a website builder (like Wix or Squarespace), or a parked page, GoDaddy may already have one or more A records set on @. These must be removed — leaving them alongside the new GitHub IPs will cause unpredictable routing and DNS errors.

Scroll through your DNS records and look for any rows where Type = A and Name = @. For each one you find, click the pencil (✏️) icon to edit or the trash (🗑) icon to delete it.

Add a CNAME Record for "www"
GoDaddy DNS — optional but strongly recommended

(Optional) Add AAAA Records for IPv6
GoDaddy DNS — improves speed for modern networks

IPv6 records are optional but recommended for future-proofing and faster load times on modern networks. Add these 4 AAAA records, same way as the A records:
